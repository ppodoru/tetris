# 🎮 Modern 1v1 Battle Tetris

현대적인 테트리스 가이드라인(Tetris Guideline)을 완벽히 준수하며, 고성능 AI 또는 전 세계의 플레이어와 실시간 대전할 수 있는 React 기반의 웹 테트리스 게임입니다.

👉 **[1인용 실시간 테스트해보기 (GitHub Pages)](https://ppodoru.github.io/tetris/)**
*서버 연동 없이 간편하게 AI 및 솔로 플레이를 테스트할 수 있습니다.*

## 1. 프로젝트 개요

- **목표**: 현대적 테트리스 규칙(SRS, 7-Bag 등)이 적용된 실시간 대전 웹 게임
- **기술 스택**: React, TypeScript, Vite, Socket.IO, Node.js
- **주요 특징**:
    - 가비지(Garbage) 공격 및 상쇄 (Combo, B2B, T-Spin)
    - 룸 리스트 기반의 간편한 실시간 1:1 대전 (Socket.IO)
    - 닉네임 설정 및 승패 기록 기반 리더보드 (JSON 퍼시스턴스)
    - 커스텀 가능한 키 바인딩 설정 저장 (localStorage)
    - 휴리스틱 기반 고성능 AI (Pierre Dellacherie 알고리즘)

## 2. 실시간 멀티플레이 (1v1 Multiplayer)

### 2.1 룸 시스템 및 닉네임
- **닉네임 설정**: 게임 시작 전 닉네임을 설정할 수 있으며 브라우저에 자동 저장됩니다.
- **방 생성**: 버튼 하나로 대기방을 생성하고 상대를 기다립니다.
- **방 참가 목록 (Room List)**: 접속 가능한 방 리스트와 방장 이름이 실시간으로 표시되며, 원클릭으로 참가할 수 있습니다.

### 2.2 🏆 리더보드
- 원격 대전(VS PLAYER) 결과가 서버에 자동으로 영구 기록됩니다 (`leaderboard.json`).
- 시작 메뉴에 인라인으로 표시되며, 상위 3명과 본인의 현재 순위를 확인할 수 있습니다.
- 승률(%) 기준으로 정렬되며, 금·은·동 메달 순위가 강조 표시됩니다.
- **전적 초기화**: `leaderboard.json` 내용을 `{}`로 바꾸고 서버를 재시작하면 초기화됩니다.

### 2.3 실시간 게임 동기화
- 블록 조작, 낙하, 가비지 전송, 재시작, 게임 오버 결과가 모두 지연 없이 동기화됩니다.
- **접속 종료 감지**: 대전 중 상대방이 나가면 "OPPONENT LEFT" 메시지와 함께 메뉴로 돌아옵니다.

## 3. 핵심 게임 메커니즘

### 3.1 현대적 테트리스 가이드라인 준수
- **SRS (Super Rotation System)**: 블록 회전 시 벽 차기(Wall Kick) 오프셋 테스트 적용
- **7-Bag System**: 7개 블록이 한 묶음으로 무작위 생성되어 편중 방지
- **Hold & Ghost Piece**: 현재 블록을 보관하고 교체 / 낙하 위치 시각적 표시
- **Lock Delay**: 바닥에 닿은 후 500ms 동안 조작 가능 (인피니티 방지)

### 3.2 공격 시스템
- **T-Spin 감지**: 좁은 공간에서 T-블록 회전 배치 시 강력한 공격력 부여
- **Tetris & B2B**: 4줄 동시 제거 시 더 많은 가비지 블록 전송, 연속 적용 시 공격력 강화
- **Combo**: 연속 라인 클리어 시 공격력 누적 증가
- **Perfect Clear**: 보드를 완전히 비우면 강력한 +10 라인 공격

## 4. 시스템 아키텍처

### 4.1 주요 모듈
| 파일 | 역할 |
| :--- | :--- |
| `src/engine.ts` | 게임 물리 로직, 충돌 검사, 가비지 계산, 라인 클리어 판정 |
| `src/ai.ts` | 보드 높이, 구멍(Holes), 울퉁불퉁함(Bumpiness) 평가 기반 최적수 탐색 |
| `src/components/TetrisBoard.tsx` | Canvas API 기반 60fps 렌더링 및 이펙트 |
| `src/App.tsx` | 메인 컨트롤러 UI 및 Socket.IO 클라이언트 로직 |
| `server.js` | Socket.IO 매칭, 룸 리스트 관리, 리더보드 데이터 영구 저장 및 중복 기록 방지 |
| `vite.config.ts` | `/socket.io` 경로 프록시 설정 (단일 포트 외부 개방용) |
| `leaderboard.json` | 플레이어 승패 기록 저장 파일 (초기화: `{}` 로 덮어쓰고 서버 재시작) |

## 5. 조작 가이드 (설정 가능)

| 기능 | 기본 키 |
| :--- | :--- |
| **이동** | `←` / `→` (DAS/ARR 적용) |
| **시계 방향 회전** | `↑` |
| **반시계 방향 회전** | `Z` |
| **소프트 드롭** | `↓` |
| **하드 드롭** | `Space` |
| **홀드** | `Shift` / `C` |

> 시작 화면 → [⚙️ Settings] 버튼에서 모든 키를 재지정할 수 있습니다. 변경된 설정은 브라우저에 자동 저장됩니다.

## 6. 실행 방법

```bash
# 의존성 패키지 설치
npm install

# 터미널 1: 게임 서버(Matchmaking & Leaderboard) 시작 (포트 3001)
node server.js

# 터미널 2: 프론트엔드 개발 서버 실행 (포트 5173)
npm run dev
```

브라우저에서 `http://localhost:5173` 접속 후 **[CREATE ROOM]** 혹은 **[JOIN ROOM]**을 통해 대전을 시작하세요!

Vite 프록시 설정을 통해 외부에 포트 `5173`만 열어두면 원격 접속 환경에서도 소켓이 자동으로 연동됩니다.
